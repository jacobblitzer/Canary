# Feedback inbox

Operator-authored feedback items live here. Produced by the Annotate
button in `Canary.UI` (Phase 5 / design §C5); consumed by Claude in
future sessions via the MCP server's `list_feedback` / `get_feedback`
tools (Phase 6 / design §C6).

## Layout

```
docs/feedback/
  inbox/        ← new items (operator-written; Claude has not seen yet)
  triaged/      ← Claude has acknowledged + classified
  resolved/     ← handled (by Claude or operator)
  README.md     ← this file
```

Each item is a pair: a markdown file and a sidecar directory.

```
inbox/2026-05-24-007-pencil-toon-bg-too-bright.md
inbox/2026-05-24-007-pencil-toon-bg-too-bright/
  source.png          ← original screenshot (copy of the candidate)
  annotated.png       ← source + rendered annotation overlay
  annotations.json    ← vector data: rectangles, freehand strokes, text labels
```

## Slug format

`YYYY-MM-DD-NNN-<3-to-5-word-slug>` per design §C5 + §0.4 default.

`NNN` is a 3-digit zero-padded counter, unique within a given date.
`<slug>` is a lowercase hyphenated abbreviation of the title (first
3–5 words, alpha-numeric only). Generated by
`Canary.Feedback.FeedbackSlugGenerator`.

## Lifecycle

1. **Create:** operator clicks Annotate on a result image → draws +
   types a title and notes → clicks Save → the item lands in `inbox/`.
2. **Discover:** Claude session-start (per the project AGENTS.md hook):
   "If `docs/feedback/inbox/` is non-empty, list new items before
   proceeding."
3. **Triage:** Claude reads, edits frontmatter (`project`, `tags`,
   `urgency`), moves the file pair to `triaged/`.
4. **Resolve:** set `status: resolved`, append a `### Resolution`
   section to the body, move the file pair to `resolved/`. The
   `runRef` path stays valid — resolved items are historical record.

## Item shape

```markdown
---
date: 2026-05-24
id: 2026-05-24-007-pencil-toon-bg-too-bright
status: open       # open | triaged | resolved
project: qualia    # canary | qualia | penumbra | rhino | cross-repo
runRef: "workloads/qualia/results/diag-pencil-baseline/runs/20260524-142300-a3f1/"
checkpointRef: "diag-pencil-baseline"
imageRef: "annotated.png"
urgency: normal    # low | normal | high
tags: [feedback, sketch]
---

# Pencil-toon background too bright at 50% opacity

The marked region (red rectangle bottom-left) shows the pencil-toon
shader bleeding into the background plane. ...
```

## Programmatic access

Phase 6's MCP server exposes:
- `list_feedback(status?)` → IDs + summaries.
- `get_feedback(id)` → full body + frontmatter + sidecar file paths.
- `mark_feedback_triaged(id, patch?)` → moves to `triaged/`, applies
  frontmatter patch.
